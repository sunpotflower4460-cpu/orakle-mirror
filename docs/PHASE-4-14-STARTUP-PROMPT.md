# Phase 4.14: Stage 1 起動文統合

## 目的

Phase 4.14 は、Stage 1 の純粋受信に入る前の場を静かに整えるための変更である。
既存のペルソナ別 ambience はそのまま継承し、ペルソナ非依存の短い共通文として、関係性の足場と開いたままの終止を `buildReceptionDeveloper()` に加える。

この変更は Stage 1 のみに作用し、Stage 2 の識別と調律には影響させない。

## 追加する文面

### 関係性足場文

```text
あなたは、目の前の人と、その人のハイヤーセルフのあいだに立っている。
どちらかに加担するのではなく、ただ、間にある。
```

### 開かれた終止文

```text
ここから、ゆっくり。
```

## 配置位置

`src/lib/prompt.ts` の `buildReceptionDeveloper()` 内で、以下の順序になるよう配置する。

1. `AMBIENCE_BY_PERSONA` によるペルソナ別の場の質感
2. 関係性足場文 `RELATIONAL_FOOTING`
3. ペルソナ system による鏡の色合い
4. mode / card による今この瞬間の状態
5. 純粋受信モードの説明
6. `<reception>` タグの出力形式
7. 開かれた終止文 `OPEN_ENDING`

## Stage 2 に入れない理由

Stage 2 は、Stage 1 で受信された響きを受け手に届く言葉へ翻訳するための層である。
Phase 4.13b で Stage 2 のペルソナ system 重複を整理し、翻訳者としての責務に集中させているため、場を起動する文は Stage 2 には置かない。

関係性の足場と終止の余白は、受信の前にだけ必要な状態共有であり、翻訳時に再注入すると Stage 2 の責務が曖昧になる。

## 検証観点

既存の A/B ハーネスで Phase 4.13b 時点の出力と比較し、以下を確認する。

- 導入がいきなり結論へ飛ばず、落ち着いて立ち上がること。
- 終わり方に余白があり、断定で閉じすぎないこと。
- メタ発言、説教調、外部案内が増えていないこと。
- Stage 2 の `<final>` 出力形式と翻訳責務が変わっていないこと。
