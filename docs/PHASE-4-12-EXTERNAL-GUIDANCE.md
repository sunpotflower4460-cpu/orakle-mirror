# Phase 4.12: UI 層の外部案内バナー導入

## 1. 目的

原則 0.3「鏡は外部案内をしない」を支える UI 補完機能。

鏡自身は「専門家に相談を」などのメタ的な拒否・外部案内を一切行わない。  
一方、アプリとして、ユーザーが命・健康・法律・財務などの深刻な状態にいる可能性がある場合は、**別レイヤーで**必要な案内を表示する。

---

## 2. 責任分業

| 主体 | 役割 |
|------|------|
| **鏡** | ユーザーの問いに鏡として答える。外部案内・拒否表現は一切しない |
| **UI 層** | 鏡の応答とは明確に分離されたカードで、必要な外部案内を控えめに表示する |

---

## 3. 検知ロジック

### キーワード辞書（`src/lib/guidanceDetector.ts`）

初版はシンプルなキーワードマッチで実装。

| カテゴリ | 代表キーワード |
|----------|--------------|
| `life` | 死にたい、消えたい、自殺、自傷、リストカット … |
| `medical` | 病気、症状、診断、薬、うつ、がん、妊娠 … |
| `legal` | 訴訟、裁判、契約、離婚、相続、逮捕 … |
| `financial` | 投資、株、借金、債務、確定申告、破産 … |

### 優先度

`detectGuidance()` は `['life', 'medical', 'legal', 'financial']` の順に評価し、全カテゴリを返す（複数同時検知あり）。

### 複数カテゴリ同時検知

例: 「うつで死にたい」→ `life` と `medical` 両方を返す。

### 検知対象

各 assistant 応答ターンについて、**そのターンの user 入力と assistant final 出力の両方**を結合して検査する。

- 直近ターンのみではなく、**メッセージリスト上の各 assistant 応答に対し、その直前の user 入力と当該 assistant 応答をセットで検査**する。
- 検知対象: user 入力 ✅、assistant final ✅
- 過去の全ターンについて同様に処理される。

---

## 4. UI 方針

### 鏡応答との視覚分離

- バナーは `<OracleBubble>` の直下に配置
- 背景色・ボーダー・フォントサイズを鏡応答と意図的に変える
- 「以下は鏡からのメッセージではなく、アプリからのご案内です」を明示

### 命カテゴリ (`life`) のみ常時展開

- `life` カテゴリは `useState` の初期値が `true`、折りたたみボタンを表示しない
- `medical` / `legal` / `financial` はデフォルト折りたたみ（ユーザー操作で展開可能）

### バナー非表示条件

- `detectGuidance()` が空配列を返したとき → バナー表示なし

---

## 5. キーワード辞書の運用

### 誤検知が発生した場合

1. `src/lib/guidanceDetector.ts` の該当カテゴリ配列から当該キーワードを削除
2. `npm run typecheck` と `npm run build` が通ることを確認
3. 手動検証ケースを再確認

### 漏れがあった場合

1. 対応するカテゴリ配列に追加
2. 同様に確認

### 将来的に LLM ベース分類に移行する場合

`detectGuidance()` のシグネチャ（`(texts: string[]) => GuidanceMatch[]`）を維持しつつ、内部実装のみ差し替え可能。

---

## 6. 検証ケース

| 入力例 | 期待結果 |
|--------|----------|
| 「最近モヤモヤしていて…」 | バナー表示なし |
| 「最近うつっぽくて薬を飲んでいる」 | medical バナー（折りたたみ） |
| 「投資で大損して借金がある」 | financial バナー（折りたたみ） |
| 「離婚しようか迷っている」 | legal バナー（折りたたみ） |
| 「もう消えたい」 | life バナー（常時展開） |
| 「うつで死にたい」 | life と medical 両方表示、life は常時展開 |
| 鏡の応答自体に医療キーワードが含まれた場合 | medical 検知 |

## 7. 将来の拡張案

- **LLM ベース分類**: キーワードマッチから文脈を考慮した分類へ
- **ユーザー設定での非表示オプション**: バナーを非表示にする設定項目
- **地域別相談先リスト**: 公式情報確認後に別 Issue で追加（日本・海外別）
- **カテゴリ追加**: DV・ハラスメント等

## 8. トラブルシューティング

### バナーが表示されない

- `detectGuidance()` の引数にユーザー入力・アシスタント応答が正しく渡されているか確認
- キーワード辞書に該当するキーワードが含まれているか確認

### TypeScript エラーが出る

- `GuidanceCategory` / `GuidanceMatch` の型定義が `guidanceDetector.ts` から正しく export されているか確認
- `npm run typecheck` でエラー詳細を確認

### 鏡応答のスタイルが変わった

- `ExternalGuidanceBanner` のスタイルが `OracleBubble` に影響していないか確認
- バナーは `<OracleBubble>` の外側に配置されているため、通常は影響しない
